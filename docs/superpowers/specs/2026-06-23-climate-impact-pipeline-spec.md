# 기후 리스크 영향 분석 — 파이프라인 작업 스펙 (logisight 레포)

> **대상 레포:** `iskkang/logisight` (데이터·뉴스·AI 생성 파이프라인). 이 프론트(`logisight-core`)는 산출물 **표시·검수**만 담당한다(Phase-6).
> **배경:** 2026-06-23 `/climate` 재설계 중 발견한 데이터 공백. 프론트에서는 실 이벤트를 노선/자산에 **지리적으로 연결(표시)**하는 선까지만 처리했고, 아래는 파이프라인이 데이터·점수·AI 산출물을 만들어야 해결된다.

## 현재 데이터 실측 (2026-06-23 anon 조회)

- `assets`: 43개. `type` = port / choke / **rail**. 그러나 **rail 자산은 전부 유라시아 TCR**(시안·호르고스·도스틱·카스·오시·알마티·타슈켄트·자바이칼스크·브레스트). **미국 내륙 철도/트럭 거점 자산이 없음.**
- `asset_risk`: 자산×4시점(0/3/7/14일) score·level·driver·wind_gust·wave_height 등. **활성 이벤트가 점수에 반영되지 않음**(예보장 기반 점수만).
- `routes`: 5개. 아시아–유럽(**수에즈 경유** — 희망봉 우회 노선 없음), 중동–아시아(에너지), 환태평양, 아시아–미동안, 환대서양.
- `events`: ~30건. 좌표 보유 23건. **NWS(미국) 중심** — 홍수(NJ/PA/MS/AR/FL)·토네이도 + 마다가스카르 가뭄. 단 **`cyclone` 1건(북서태평양, 125.6E/18.5N, severity r)** 존재.

## 발견된 공백 → 작업 항목

### 1. 서태평양 열대저기압 이벤트 커버리지
- **문제:** 이벤트 소스가 미 NWS 위주. 동아시아(대만·한·중·일) 태풍이 거의 안 들어옴. 현재 들어온 `cyclone` 1건도 북서태평양 광역 좌표라 노선 연결이 약함(대만 해협까지 ~880km).
- **작업:** GDACS-TC / **JTWC** / **JMA(기상청 RSMC Tokyo)** / **HKO** 트랙 데이터를 `events`에 적재. 태풍은 단일 핀이 아니라 **예보 트랙(폴리라인) + 반경/강도**를 저장해 노선 교차 판정이 가능하게 한다.
- **스키마 제안:** `events`에 `track jsonb`([[lon,lat,radius_km,intensity], …], 시점별), `valid_at timestamptz`, `kind='cyclone'` 표준화.

### 2. 이벤트 → asset_risk 전파 (핵심)
- **문제:** `events`와 `asset_risk`가 독립. 활성 태풍/홍수가 인근 자산·노선 점수를 올리지 않아 "영향 노선"이 예보장 점수만 반영. (프론트는 임시로 거리 기반 표시만 함.)
- **작업:** 스코어링 잡에서 **활성 이벤트의 자산/노선 근접·강도를 asset_risk.score·driver에 가산**. 예: 자산 반경 R 내 severity=r 이벤트 → score += 강도×감쇠, driver=이벤트 종류. 노선 점수는 자산 점수 max 유지.
- **결과:** 프론트의 거리-표시 임시 로직 없이도 "영향 노선"이 정합. (프론트 임시 로직은 파이프라인 반영 후 제거 가능.)

### 3. 미국 내륙 철도·트럭 거점 자산 추가
- **문제:** 미동부 홍수(#사용자 보고)가 **내륙 철도/트럭 지연**으로 연결 안 됨 — 미 내륙 intermodal 자산 자체가 없음.
- **작업:** `assets`에 미 주요 intermodal/rail hub·트럭 코리도어 추가(예: Chicago, Memphis, Dallas, Kansas City, Atlanta, NJ/NY 항만 연계 내륙거점), `type='rail'` 또는 신규 `type='inland'`. 해상 노선의 **내륙 연결 구간(post-port)** 을 표현할 노선/세그먼트도 검토.
- **결과:** NJ/PA·중남부 홍수가 인근 내륙 거점 리스크로 표면화 → 화주에게 "항만 통관 후 내륙 구간 지연" 신호 제공.

### 4. 희망봉 우회 아시아–유럽 노선 추가
- **문제:** `routes`에 희망봉 경유 노선이 없음. 홍해/수에즈 리스크 국면에서 **실제 주력 우회 항로**가 빠져 있음.
- **작업:** `routes`에 "아시아–유럽 (희망봉 우회)" 추가 — waypoints: 말라카→콜롬보→인도양→희망봉(goodhope)→남대서양→지브롤터→로테르담. `chokes` 갱신. (프론트 CapeMonitor는 이미 희망봉 경유 자동 감지 → 노선 추가 시 "희망봉 우회 항로 모니터링"으로 자동 전환.)

### 5. AI 영향 분석 산출물 (Phase-6)
- **문제:** 사용자 요구 = "AI로 더 디테일하게 영향도 분석·예상". 단 **AI 생성은 파이프라인 소관**, 프론트는 표시·검수만.
- **작업:** 주간(또는 이벤트 트리거) 스크립트 + Claude API로 **이벤트/노선별 영향 전망**을 생성해 `forecasts`(module 신규 'climate' 또는 'eurasia') `status='draft'` 적재.
  - **본문 3단 변환 강제(Phase-6):** ① 기상 리스크 변화(태풍/홍수 강도·트랙) → ② FEU/리드타임·항만 적체 영향(예: "+N일") → ③ 권장 행동 1개. **확률 표현 강제, 인과 단정 금지**(`정합/추정/상관`).
  - **에디터 검수 후에만 published**, 발행 후 본문 수정·삭제 불가, 화면에 "AI 초안 · 에디터 검수" 표기(프론트 기존 패턴 재사용).
  - 적중률은 published 전수 기준, 판정일 실측으로 사후 채점.
- **프론트 표시(후속):** `/climate` "기상 리스크 → 영향 노선" 카드/상세에 published climate forecast를 연결(현재는 결정론적 거리-표시 + risk score만).

## 우선순위 제안
1. (2) 이벤트→asset_risk 전파 — 가장 큰 정합 개선, 다른 항목의 토대.
2. (1) 서태평양 태풍 소스 — #3 근본 해결.
3. (3) 미 내륙 자산 — #4 근본 해결.
4. (4) 희망봉 노선 — 1줄 데이터 추가, 프론트 자동 대응.
5. (5) AI 영향 분석 — (1)~(3) 데이터가 받쳐줄 때 품질 확보.

## 프론트(이 레포)에서 이미 처리한 임시 대응 (2026-06-23)
- `/climate` "영향 노선"을 **이벤트-인지**로 변경: 실 이벤트 좌표 ↔ 노선 경로점 근접(≤950km) 연결, 태풍·폭풍 우선 노출, "활성 이벤트 N건 인근" 표기. (파이프라인 (2) 반영 시 단순화/대체 가능.)
- Timeline 정렬을 태풍·폭풍 우선으로 변경(다수 홍수 경보에 묻히지 않게).
- CapeMonitor: 희망봉 경유 노선이 실제 있을 때만 "우회" 표기(없으면 "유럽 주력 항로").
- 더미 수치·임의 지연일 미표기, 정성·추정 표현만 사용(Phase-6/데이터 안전 준수).
