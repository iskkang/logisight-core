# /rates 페이지 KITA 스타일 검색 재구성 — 설계

작성일: 2026-06-15
대상 레포: `logisight-core` · 대상 파일: `src/routes/rates.tsx` (+ 신규 하위 컴포넌트)

## 1. 목표

현재 `/rates`(Rates Control Tower)는 상단 필터바(운송방식 전체/해상/항공 · 기간 · 출발지 · 도착지) 아래에 **해상·항공 운임 테이블이 길게 stacked**로 노출된다. 이를 KITA(`logisticsSeaFare`) 스타일의 **검색바 → 결과(차트 + 테이블)** 구조로 전환한다.

핵심 변경:
- 운송방식 **"전체" 제거** → 분류는 **해상/항공**만.
- 긴 stacked 운임 목록 제거 → **검색 결과만** 차트·테이블로.
- 검색바를 KITA 폼처럼(분류·출발지·도착지 권역▸항만·기간 년월 범위·검색/초기화).
- **전월대비 변동률 히트맵(주요 집중 6개 노선)은 유지.**

## 2. 데이터 사실 (구현 전제)

기존 쿼리(`kitaSeaRatesQueryOptions`, `kitaAirRatesQueryOptions`)가 전 행을 클라이언트로 가져오므로 **검색·필터는 전부 클라이언트 측**에서 처리한다(신규 API 불필요).

- `kita_sea_rates`: origin(전부 **부산**), dest, region, year_mon(`YYYYMM`), teu, feu, teu_chg, feu_chg
- `kita_air_rates`: origin(전부 **인천**), dest, region, year_mon, kg100, kg300, kg500, chg100/300/500
- 권역(region) 값 — 해상: 북미·중남미·유럽·아시아·일본·중국·아프리카·오세아니아·중동·러시아/CIS. 항공도 유사(라벨 일부 상이: "러시아"). 권역마다 항만(dest) 다수(해상 5~22개, 항공 1~8개).
- **출발지는 모드당 1개**뿐 → 드롭다운은 단일값(해상 부산 / 항공 인천), 사실상 표기용.
- 이상치는 적재 단계 `rate-outlier-guard`(±150% carry-forward, logisight 레포)가 이미 차단 → 표시값 신뢰.

## 3. 검색바 (현 상단 필터바 + "전체 운임 목록" 검색 대체)

필드 순서(KITA·이미지2 기준):
1. **분류**: 해상 / 항공 (세그먼트 또는 select). 기본 **해상**. 전환 시 출발지·권역·항만·테이블 컬럼이 해당 모드 데이터로 갱신.
2. **출발지**: 해상=부산, 항공=인천. 단일 옵션 select(표기용).
3. **도착지 — 권역**: 선택 모드의 distinct region. 기본 **북미**.
4. **도착지 — 항만**: 선택 권역의 distinct dest + 맨 앞 **"전체(권역)"** 옵션. 기본 **전체(권역)**.
5. **기간**: 시작(년·월) ~ 종료(년·월) select 4개. 범위 = 데이터 min/max year_mon. 기본 종료=최신월, 시작=최신월−12개월.
6. **[검색]** / **[초기화]**. 초기화 = 위 기본값 복귀.

동작: 폼 상태를 적용해 결과(차트·테이블)를 갱신. (구현 단순성을 위해 상태 변경 즉시 반영해도 무방하나, KITA처럼 명시적 [검색] 버튼 제공.)

## 4. 결과 차트 (현 "노선 운임 추이 비교" 대체)

- **항만 선택 시**: 단일 노선(부산→항만)의 기간 내 월별 추이 1개 라인.
- **권역만(항만=전체) 시**: 권역 내 항만들의 멀티라인. **스파게티 방지 — 최신월 값 상위 8개 항만만** 라인으로(나머지는 테이블에서 확인). (상위 N은 튜닝 가능 상수.)
- **FEU/TEU 토글**(해상). 항공은 kg300 기준(토글 없음 또는 kg100/300/500 선택 — 1차 구현은 kg300 고정).
- X축 = 기간 범위 월(`YYYY.MM`), 정렬 오름차순.

## 5. 결과 테이블 (긴 stacked 목록 대체)

높이 제한(maxHeight + 스크롤)로 **다시 길어지지 않게** 한다.

- **항만 선택 시**: 그 노선의 기간 내 **월별 행**(최신월 우선). 컬럼 — 해상: 년·월·TEU·FEU(+변동); 항공: 년·월·kg100·kg300·kg500.
- **권역만 시**: **항만당 1행**(기간 내 최신월) + 전월대비(MoM). 컬럼 — 노선·최신월·TEU·FEU·MoM. MoM 또는 값 기준 정렬.
- 결과 0건이면 "검색 결과가 없습니다" 안내.

## 6. 유지 (변경 없음)

- **전월대비 변동률 히트맵**: 기존 `HEATMAP_ROUTES` 고정 6개 노선(부산→롱비치·서배너·함부르크·하이퐁·코페르·알렉산드리아). **검색과 무관하게 고정.** (좌측 차트가 검색 연동으로 바뀌므로 히트맵과는 의도적으로 분리됨.)
- **글로벌 지수 추이**: 최근 6개월(기존 수정 유지).
- **최근 리포트**: WCI·SCFI·KCCI 3개 인라인 카드(기존 유지).
- PageHero, 환율 표기.

## 7. 제거

- 운송방식 **"전체"** 모드(`MODE_KO`의 전체, `mode="all"` 분기).
- **KPI 스트립**(상승/하락/보합/이상치 `CountKpi` 4장)과 그 산출(`risingCount` 등).
- **긴 "전체 운임 목록" Panel** + `RouteGroupTable`(해상·항공 stacked) + 관련 `seaVisible`/`airVisible`/검색 input.
- 기존 상단 필터바(`FilterSeg` 운송방식/기간, `FilterSelect` 출발지/도착지, 초기화).

## 8. 컴포넌트 구조 (rates.tsx 내)

- `RateSearchBar` — 분류·출발지·권역·항만·기간·검색/초기화. 상태를 상위로 콜백.
- `RateResultChart` — 단일/멀티라인 자동 + FEU/TEU 토글.
- `RateResultTable` — 항만/권역 모드별 컬럼·행.
- 유지: 히트맵 블록, `RateReports`(최근 리포트), 글로벌 지수 추이 차트.
- 순수 헬퍼(테스트 대상): 검색 폼 → 결과 행 선택 로직
  - `seaRegions(rows)`, `portsOf(rows, region)`
  - `routeSeries(rows, origin, dest, range, metric)` — 단일 노선 시계열
  - `regionLatest(rows, region, range)` — 권역 항만당 최신월 + MoM
  - 기간 범위 필터 `inMonthRange(rows, startYM, endYM)`

## 9. 테스트 (vitest, 순수 함수 단위)

- 권역/항만 목록 추출(모드별 origin 고정).
- 기간 범위 필터 경계(시작=종료, 역전 입력 방어).
- 단일 노선 시계열 정렬·결측 처리.
- 권역 최신월 1행/항만 + MoM(직전월 대비), carry-forward 후 값 기준.
- 멀티라인 상위 N 캡.

## 10. 비목표 (Non-goals)

- 신규 API/테이블/라우트 없음(검색은 클라이언트 필터).
- 항공 kg100/300/500 토글 고도화는 차후(1차 kg300 고정).
- 히트맵 노선 동적화 없음(고정 유지).
- 출발지 다중화 없음(데이터가 단일 origin).

## 11. 리스크 / 추후 조정

- 권역 멀티라인 상위 N(8) 캡은 실제 데이터 보고 조정 가능.
- 기간 기본폭(12개월)·기본 권역(북미)은 사용 후 조정 가능.
- 사용자: "실제 데이터를 보고 수정" 예정 — 차트 가독성·테이블 정렬 기준은 구현 후 검토.
